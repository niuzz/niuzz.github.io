js基础-数据类型

发表于 2019-09-27 分类于 javascript 阅读次数：

数据类型是 js 基础的第一篇，也说明了数据类型是基础内容中的基础。

尽管概念很简单，但是因为JavaScript语言特点，实际上有很多不符合我们理解的现象存在。而理解这一切的基础，在于熟悉规则。

每一部分基础内容都会由知识点展开相关内容。

  • 数据类型分类

0x01 数据类型分类

JavaScript 中共有七种内置数据类型，包括基本类型和对象类型。

基本类型

基本类型分为以下六种:

  • string（字符串）
  • boolean（布尔值）
  • number（数字）
  • symbol（符号）
  • null（空值）
  • undefined（未定义）

其中symbol是ES6中新增的数据类型。

对象数据类型

对象、function、array是对象类型，对象值是存储的引用地址。

0x02 类型转换

类型转换在js中非常容易发生隐藏的bug, 同时也是面试高频考点。

因为涉及的规则众多，所以记忆它的方式显得尤为重要。先主后次，理解实践
把使用最多的string，boolean，number的转换规则当作重点，有针对的加以练习。

toPrimitive

当操作符两边的类型不一致时，需要强制类型转换，会调用内部的toPrimitive函数

可选参数PreferredType是Number或者是String。返回值为任何原始值

如果preferedType是string

  1. input 是基本类型，就返回这个值。
  2. 否则，input为Object。调用 obj.valueOf()。如果结果是primitive，返回。
  3. 否则，input为Object。调用 obj.toString()。如果结果是primitive，返回。
  4. 否则，抛出TypeError。

如果PreferredType是number

  1. input 是基本类型，就返回这个值。
  2. 否则，input为Object。调用 obj.toStringf()。如果结果是primitive，返回。
  3. 否则，input为Object。调用 obj.valueOf()。如果结果是primitive，返回。
  4. 否则，抛出TypeError。

如果type为空

  1. 该对象为Date，则type被设置为string
  2. 否则，type被设置为number

其中valueOf是把对象转换成原始类型的值，不同内置对象的实现

Number 运算符转换

  • null转换为0
  • undefined转换为NaN
  • true转换为1， false转换为0
  • 字符串转换时遵循数字常量规则，失败返回NaN
  • 对象转换成原始值 type 指定成 number

String 运算符转换

  • null转换为‘null’
  • undefined 转换为 undefined
  • true 转换为 ‘true’， false 转换为 ‘false’
  • 数字转换遵循通用原则，极大极小使用指数形式
  • 对象转换成原始值 type 指定成 string

Boolean 运算符转换

除了下述 6 个值转换结果为 false，其他全部为true：

  1. undefined
  2. null
  3. -0
  4. 0或+0
  5. NaN
  6. ‘’（空字符串）

自动转换

数据类型将在一些特定场景下，发生自动转换。

  • +运算
  • ==运算
  • 条件判断

string自动转换

string的自动转换，主要发生在字符串的加法运算，当一个值为字符串另一个值为非字符串，则转为字符串拼接。

如果和对象进行相加，则看对象toString是否有返回，如有是字符串与返回值的拼接，没有则输出字符串+对象toString方法返回的字符串。

number自动转换

加法运算符，无string的时候会优先转换为Number

==抽象相等，Number优先

需要注意的是，对象 toPrimitive() type 为number进行转换， 存在boolean或者string type 为number转成 0，1 进行比较
